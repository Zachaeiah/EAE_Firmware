# EAE Firmware PID Simulation

This project demonstrates a discrete PID motor-position control loop for the EAE firmware challenge.

The project includes **two PID simulation demos**:

1. **Floating-point PID simulation**
2. **Fixed-point PID simulation**

Both demos simulate the same closed-loop motor-position control system. The difference is the math method used inside the controller and plant simulation.

The floating-point version uses normal `float` math. The fixed-point version uses scaled integer math to demonstrate how the same type of control loop could be implemented on embedded hardware where floating-point math may be slower or unavailable.

The simulation runs at **1 kHz** for **1 second**.

Both the controller and motor plant are implemented as discrete Z-domain transfer functions.

---

## Project Overview

This project is used to design, simulate, log, and plot a closed-loop PID motor-position control system. The C program runs the PID simulation and saves the results to a CSV file. The Python tools are used to design the controller, validate the system response, and plot the simulation output.

The goal is to show a firmware-style control loop running on a PC without needing real motor hardware.

---

## Control System Description

The simulated control loop is:

```text
Position Setpoint
      |
      v
   [ Error ] = Setpoint - Measured Position
      |
      v
[ PID Controller C(z) ]
      |
      v
Voltage Command
      |
      v
[ Motor Plant Gp(z) ]
      |
      v
Measured Position
      |
      +---- feedback to controller
```

The controller input is the position error. The controller output is a simulated motor voltage command. The motor plant takes that voltage command as its input and outputs the simulated motor position. That position is fed back into the controller on the next update.

## Two Simulation Demos

This project includes two versions of the same PID simulation.

### Floating-Point Demo

The floating-point demo uses normal `float` values for the controller, plant, error calculation, and voltage command.


### Fixed-Point Demo

The fixed-point demo performs the same type of PID control, but uses scaled integer math instead of normal floating-point math.

## C Simulation

The main simulation is run from:

```text
build/main.exe
```

This executable runs the closed-loop PID simulation. During each loop update, the program calculates the position error, updates the PID controller, limits the voltage command, updates the motor plant, and records the result. This is meant to represent the type of logic that could later run in embedded firmware, but it runs on a PC for easier testing.

---

## Simulation Output

The simulation saves its results to:

```text
output/pid_fixed_output.csv
output/pid_floating_output.csv
```

This CSV file stores the data generated by the PID loop. It can be used to review the system response after the simulation finishes. The output includes values such as time, setpoint, measured position, position error, raw voltage command, and clamped voltage command.

---

## Python Plotting Script

The simulation output is plotted using:

```text
tools/control_design/plot_sim.py
```

This script reads the CSV file from `output/pid_output.csv` and plots the results. The plots are used to check how well the motor position follows the setpoint, how large the error is, and how much voltage the controller commands during the simulation.

---

## Python Model Design Script

The controller and plant are designed using:

```text
tools/control_design/modle_design.py
```

This script is used to validate the motor plant and PID controller before copying the final discrete transfer-function coefficients into the C simulation. It is used to inspect the step response, poles, root locus, and general closed-loop behavior of the system.

The final discrete Z-domain transfer-function coefficients from this script are copied into the C simulation.

## How the Simulation Works

The simulation runs with a sample time of:

```text
0.001 seconds
```

This is equal to:

```text
1 kHz
```

The full simulation runs for:

```text
1 second
```

At each update, the controller compares the desired motor position to the simulated measured position. The difference becomes the error signal. The PID controller uses this error to calculate a voltage command. The voltage command is then passed into the motor plant transfer function, and the plant outputs the next motor position.

---

## Floating-Point vs Fixed-Point

The floating-point and fixed-point demos follow the same control-loop structure:

```text
setpoint -> error -> PID controller -> voltage command -> motor plant -> measured position
```

The floating-point demo stores values directly as decimal numbers.

The fixed-point demo stores values as scaled integers. For example, a real value such as `1.5` may be stored internally as `1500` if the scale factor is `1000`. This lets the program do decimal-style math using integer operations.

The fixed-point version is more sensitive to scaling, overflow, and rounding. This makes it a useful demonstration of embedded control-system implementation details.

## Purpose of the Project

This project demonstrates a complete control-system workflow. The motor is represented by a discrete plant transfer function, and the PID controller is also represented by a discrete Z-domain transfer function. This allows the controller to be tested and tuned on a PC before applying the same control-loop idea to real embedded firmware.

The project shows:

- discrete PID control
- motor plant simulation
- closed-loop feedback
- voltage command limiting
- floating-point control math
- fixed-point control math
- CSV data logging
- Python plotting
- controller validation before firmware use

---

## Notes

This simulation does not require real motor hardware.

The voltage command is simulated. It represents what the controller would send to a motor driver in a real system.

The motor position is also simulated. It is calculated by the discrete motor plant transfer function.

The floating-point and fixed-point demos are intended to do the same thing, but with different math implementations. The floating-point version is simpler to understand, while the fixed-point version demonstrates a more embedded-style approach.
